# 产品需求文档 (PRD)

## 1. 项目背景

大学党员工作站需要一套排班管理系统，替代手工排班。由管理员录入成员课表后，系统自动生成值班安排，支持请假替补、统计导出等功能。

## 2. 用户角色

系统只有一种角色：**管理员**。所有使用者（学生负责人、指导老师等）拥有同等权限，数据互通。

## 3. 人员层级与排班优先级

| 层级 | 人数 | 排班优先级 | 说明 |
|------|------|-----------|------|
| 部员 | 40-50人 | 第1优先 | 值班主力 |
| 部长 | 12-16人 | 第2优先 | 部员不足时启用 |
| 主席团（站长） | 5人 | 第3优先 | 最后兜底 |

## 4. 值班时段

仅限**工作日（周一至周五）**，每天3个时段：

| 时段 | 课节 | 时间 |
|------|------|------|
| 上午段 | 第3-4节 | 9:30 — 11:00 |
| 下午段A | 第6-7节 | 14:00 — 15:30 |
| 下午段B | 第8-9节 | 15:30 — 17:00 |

## 5. 核心业务规则

### 5.1 课表驱动
- 某人某时段有课 → 不可值班
- 某人某时段没课 → 可值班
- 课表按单周/双周分别录入（因单双周课程不同）

### 5.2 单双周
- 按学校教学周历判断
- 管理员在学期初设置第一周是单周还是双周

### 5.3 排班模式（管理员手动切换）
- **一般模式（默认）**：尽量不安排同一人连续两周值班
- **紧急模式**：允许连续安排，确保每天每时段都有人

### 5.4 排班优先级算法
1. 角色优先：部员 > 部长 > 主席团
2. 公平优先：同等角色内，历史值班次数少的优先
3. 连续性约束（一般模式）：排除上周已值过班的人
4. 能力不足时逐级放宽：先放宽连续性 → 再提升角色层级

### 5.5 每日最低保障
- 优先保证每天至少有1人值班
- 在此基础上再追求每时段都有人
- 实在没人也接受（标记缺人）

### 5.6 请假替补
- 管理员对排班表中某人标记"请假"
- 系统自动找合适替补（同规则筛选）
- 被请假者标记为"下周仍需安排"
- 下周排班时优先补排此人

### 5.7 每时段人数配置
- 管理员可设置每天每个时段需要几个人
- 默认每个时段1人

## 6. 功能清单

| 编号 | 功能 | 说明 |
|------|------|------|
| F1 | 管理员登录 | 账号密码登录 |
| F2 | 学期管理 | 设置学期名、起始周属性、总周数 |
| F3 | 成员管理 | 增删改查、角色设置、批量导入 |
| F4 | 课表管理 | 按人+单双周录入课表、网格交互 |
| F5 | 时段配置 | 设置每天每时段需要几人 |
| F6 | 自动排班 | 选择周次，一键生成排班 |
| F7 | 手动微调 | 点击格子更换值班人 |
| F8 | 请假处理 | 标记请假、推荐替补、下周补排 |
| F9 | 模式切换 | 一般/紧急模式一键切换 |
| F10 | 排班查看 | 周视图展示，按周切换 |
| F11 | 值班统计 | 按周/学期汇总值班时长 |
| F12 | 数据导出 | 导出Excel（成员值班时长） |
