# 党员工作站排班系统 — 使用手册

---

## 📌 系统概述

本系统为大学党员工作站设计的Web排班管理工具，支持成员管理、课表录入、自动排班、请假替补、值班统计等全流程工作。

**访问地址**：`https://JX0523.github.io/party-station-scheduler/`

**适用规模**：5名主席团 + 12-16名部长 + 40-50名部员（共约60-70人）

---

## 🚀 快速开始

### 1. 注册/登录

1. 打开系统网址，进入登录页面
2. 首次使用需由管理员在 Supabase 后台手动创建账号
3. 输入邮箱和密码登录

### 2. 首次配置流程（按顺序操作）

| 步骤 | 页面 | 操作 |
|:--:|------|------|
| ① | **学期设置** | 设置学期名称、总周数、当前周、首周单/双周 |
| ② | **成员管理** | 录入/导入所有成员（姓名+角色） |
| ③ | **课表录入** | 为每位成员录入单双周课表 |
| ④ | **排班管理** | 自动生成排班（首页会自动生成当前周） |

---

## 📄 页面功能详解

### 一、首页仪表盘（Dashboard）

系统的主操作界面，分为三个区域：

#### 信息卡片区（顶部）
显示当前学期、当前周数、成员数、课表数、已完成周数

#### 本周值班表（中部，大表）
- 显示当前周的值班安排（5天 × 3时段）
- **橙色高亮**：当天所在的列
- 每格显示值班人姓名和角色
- **操作按钮**：
  - `请假` — 标记某人请假，不计时长，下周补排
  - `+ 加人` — 临时为某时段添加值班人员
- 点击 **🔄 重新生成** 重新排当前周

#### 历史值班表（底部，小表网格）
- 展示所有已完成周（上周及之前）的排班
- 紧凑卡片形式，只读不可修改
- 请假标记为 `假` 橙色标注

### 二、学期设置

| 配置项 | 说明 |
|--------|------|
| 学期名称 | 如"2026-2027学年秋季学期" |
| 总周数 | 学期教学总周数（默认20周） |
| 当前是第几周 | 设定当前周 → 之前的周自动锁定 |
| 第一周是单/双周 | 按学校教学周历设置 |

> ⚠️ **重要**：`当前周`的数值决定了哪些周的排班被锁定。第N周及之前不可修改。每周结束后请更新。

### 三、成员管理

- **添加成员**：输入姓名，选择角色（部员/部长/主席团）
- **编辑成员**：点击修改姓名或角色
- **删除成员**：删除不需要的成员
- **批量导入**：支持 Excel 导入

### 四、课表录入

- 选择成员 → 选择单周/双周 → 在网格中点击切换有课/无课
- **绿色格子** = 该时段没课，可以值班
- **红色格子** = 该时段有课，不能值班
- 支持一键复制课表到另一周类型

> 💡 每位成员需要录入**单周**和**双周**两份课表（因课程安排可能不同）

### 五、排班管理

#### 选择周次
- 顶部选择要排班的周数
- 自动显示该周是单周还是双周
- 历史周显示 **🔒 已锁定**，不可修改

#### 时段人数调整
- 点击表头中的数字，可修改该时段需要的人数
- 默认每时段1人，可按需调整（如某时段需要2人）

#### 自动生成排班
- 点击 **🔄 自动生成排班** 一键排班
- 算法自动考虑：课表冲突、上周已排人员、角色优先级、历史公平性
- 展示生成结果及角色参与情况

#### 请假操作
1. 在排班表中点击某人旁边的 `请假` 按钮
2. 确认后系统打开替补选择弹窗
3. 从候选列表中选择替补人员
4. 被请假人：本次不计时长，下周自动补排

#### 锁定机制
- 当前周之前的排班**自动锁定**
- 锁定后不可重新生成、不可请假修改
- 请前往统计页面查看和导出数据

### 六、统计导出

- **按周查看**：每位成员的值班时长和请假次数
- **学期汇总**：总值班时长、请假人次、已值班人数
- **Excel 导出**：一键导出统计数据

> 请假的值班时长为 **0**，只记录请假次数（不计时长）。

## 🧠 排班算法规则

| 规则 | 说明 |
|------|------|
| **角色优先级** | 部员为主力（~80%时段），部长偶尔参与（每周≤3人），主席团仅在人力不足时启用 |
| **课表驱动** | 某时段有课的人不参与该时段排班 |
| **不连续值班** | 同一人不会连续两周值班（保证休息） |
| **历史公平** | 历史排班次数少的人优先安排 |
| **跨周均衡** | 优先安排"双周课多、单周空闲"的人在单周值班，把"两周都空"的人留给双周 |
| **每日覆盖** | 优先保证每天至少1人值班 |
| **自然空缺** | 所有人都没时间时，该时段留空，不强制填充 |
| **补排机制** | 请假后下周自动优先安排该人值班 |

## 📊 常见操作流程

### 每周例行操作

```
① 打开首页 → 查看本周值班表
② 如有请假 → 点击「请假」→ 选择替补
③ 如需加人 → 点击「+ 加人」→ 选择人员
④ 本周结束后 → 学期设置中把「当前周」+1
⑤ 系统自动为新一周生成排班
```

### 新学期开始

```
① 学期设置 → 更新学期配置
② 成员管理 → 更新成员名单（毕业生移除、新生加入）
③ 课表录入 → 录入学新课表（或清空重录）
④ 首页自动生成第1周排班
```

---

## ⚠️ 注意事项

1. **课表必须先录入**，否则算法认为该成员所有时段都空闲
2. **当前周更新**后，之前周的数据锁定，确保数据一致性
3. **主席团极少参与排班**（仅在部员+部长总数不足5人时），这是正常设计
4. **空位表示**该时段无人可用（所有人都有课），管理员可考虑调整时段配置
5. 系统支持多台电脑同时访问，数据实时同步
